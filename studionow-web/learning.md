## Recurring writer feedback

<!-- Add patterns you notice from the feedback log. Format: what the tool does wrong → what it should do instead. -->

- Chaptered/pillar scripts feel isolated. Each section drops in cold with no connective tissue. → Write transition lines between sections. The VO should carry the viewer from one idea to the next, not just announce the next topic. Use contrast, callback, or momentum to link sections.
- The tool echoes deck language instead of translating it into script language. "Turbocharging the core" and "PresentPlay / FuturePrep / FutureProof" are strategy terms, not VO. → Rewrite strategy language into human, speakable lines. If a phrase reads like a slide title, it does not belong in VO.

## Common mistakes this tool makes

<!-- When you see the same issue show up 3+ times, add it here so the tool stops doing it. -->

- Ignoring "must-include" data points from the brief. If the brief lists specific claims, metrics, FEI values, proof points, or required language, those MUST appear in the script. Either weave them into VO or display them as animated SUPERs/data slates. Do not drop required content.
- Treating a brief's "story flow" section as a literal shot list instead of a narrative spine. The story flow gives you the arc. Your job is to write it as a script with transitions, rhythm, and momentum — not transcribe it section by section.

## Tone corrections

<!-- Specific language or tonal shifts that writers keep requesting. -->

- VO should feel like a person talking with confidence, not a deck being narrated. If a line sounds like it belongs on a PowerPoint slide, rewrite it until it sounds like something a smart person would actually say out loud.
